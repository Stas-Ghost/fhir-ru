Валидация ресурсов

[%wgt fhir%] Рабочая группаУровень зрелости: N/AСтатус голосования: DSTU 2

На этой странице приведён краткий обзор того, как спецификация FHIR поддерживает валидацию ресурсов.

В принципе, ресурсы можно валидировать:

  • на соответствие исходной спецификации;
  • на соответствие руководству по реализации;
  • на соответствие конкретному профилю.

Ресурсы можно тестировать на соответствие:

  • с помощью XML Schema - только для XML-экземпляров
  • с помощью FHIR Validator - java jar-файл, предоставляемый в рамках спецификации
  • запросить FHIR Server - используя операцию $validate
  • через веб-интерфейс - используя веб-форму на тестовом сервере

Отметьте, что все эти методы валидации не являются полными; они могут валидировать вычисляемые аспекты соответствия, однако всегда есть дополнительные правила, описанные текстом, которые они не могут проверить (например такое правило, как "Всё клинически важное содержимое в данных ДОЛЖНО быть представлено в описательной части", которое может быть введено в руководстве по реализации и не может быть проверено инструментами проверки на соответствие).

В случает расхождения между этими методами проверки на соответствие, отметьте, что:

  • Схема/схематрон - наименее действенный - в основном из-за того, что он не подключен к терминологическому серверу
  • Java-валидатор хорош настолько же, как и лежащие в основе определения и, в частности, зависит от того, поддерживает ли лежащий в основе терминологический сервер все необходимые терминологии
  • В целом, операции валидации на сервере используют или являются производными от кода валидатора на java, поэтому имеют те же подводные камни
  • Последней инстанцией является проверка человеком содержимого ресурсов, а также релевантных руководств по реализации и базовой спецификации

Также обратите внимание, что статического тестирования содержимого ресурсов не достаточно для подтверждения соответствия спецификации. Дополнительную информацию см. в разделе Тестирование на соответствие FHIR.

Использование XML-схемы

Для валидации XML-представлений ресурсов можно использовать XML-схему. Для валидации ресурса вы можете использовать одну из следующих схем:

  • fhir-all.xsd: объединяет все индивидуальные модульные схемы
  • fhir-single.xsd: единый большой файл, предоставляемый главным образом для процессоров схемы, которые не поддерживают циклические ссылки

Кроме того, валидационная схема включает в себя схематрон, который можно запустить с преобразованием "iso_svrl_for_xslt2.xsl", включённым в загрузку XML-инструментов. Отметьте, что для запуска схематронов требуется XSLT2.

Запуская схематрон, используйте файл "fhir-invariants.sch". Он включает в себя все схематроны. Отдельные файлы схематронов для каждого ресурса приводятся, чтобы дать возможность реализаторам создавать их собственный комбинированный файл меньшего размера, покрывающий только требуемые им типы ресурсов.

Использование FHIR Validator

FHIR Validator - это Java jar-файл, поставляемый в рамках спецификации и используемый во время процесса публикации для валидации всех публикуемых примеров. Для запуска FHIR-валидатора выполните следующие шаги:

  • Скачайте FHIR Validator
  • Скачайте одно из определений FHIR (с или без текста)
  • Запустите валидатор, указав ему путь к определениям и ссылку на ресурс, который нужно провалидировать

Вот пример исполняемого файла для Windows, который демонстрирует этот процесс (в помощью таких общих утилит, как wget и 7z): 

 @ECHO OFF

 ECHO get the validator and unzip it 
 wget http://hl7.org/fhir/validator.zip
 7z.exe x validator.zip

 ECHO Get the validation source file (dictionary)
 wget http://hl7.org/fhir/validation-min.xml.zip

 ECHO get an example to validate
 wget http://hl7.org/fhir/patient-example.xml -O pat-ex.xml

 ECHO validate it. The DAF profile will be loaded out of the definitions in validation-min.zip
 java -jar org.hl7.fhir.validator.jar pat-ex.xml -defn validation-min.xml.zip -profile http://hl7.org/fhir/StructureDefinition/daf-patient

 pause

Обратите внимание, что не нужно предварительно скачивать сам ресурс; HTTP-адрес можно указать напрямую: 

 java -jar org.hl7.fhir.validator.jar http://hl7.org/fhir/patient-example.html -defn validation-min.xml.zip -profile http://hl7.org/fhir/StructureDefinition/daf-patient

Запросить FHIR-сервер

Операцию validate можно использовать для проверки, соответствует ли ресурс профилю. Простейший способ выполнить эту операцию - это запостить ресурс на сервер: 

 POST [base]/Patient/$validate?profile=http://hl7.org/fhir/StructureDefinition/daf-patient
 [other HTTP headers]
 
 <Patient>.. resource to check as the body

Сервер вернёт ресурс OperationOutcome со списком всех ошибок, обнаруженных в ресурсе.

Есть несколько вещей, которые необходимо учитывать при использовании этой операции:

  • Не все серверы поддерживают операцию $validate, хотя некоторые из публичных тестовых серверов поддерживают
  • Серверы поддерживают операцию $validate в основном с валидацией на соответствие только профилям, уже зарегистрированным на сервере
  • Серверы могут на выбор поддерживать XML, JSON или оба формата

Через веб-интерфейс

Некоторые серверы реализуют функциональность $validate через веб-страницу. Список известных опубликованных реализаций можно посмотреть в wiki-разделе FHIR
