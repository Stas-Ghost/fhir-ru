Другие стандарты HL7

Организацией HL7 был создан ряд других стандартов, которые не пересекаются с FHIR настолько близко, как перечисленные выше, в основном потому что они не направлены исключительно на обмен информацией. Тем не менее, они заслуживают краткого упоминания:

EHR Functional Model (EHR-FM)

Эта спецификация определяет функциональное поведение для систем электронных медицинских записей. FHIR является стандартом обмена медицинской информацией, который может использоваться для удовлетворения части этого функционального поведения. Подробнее о том, как FHIR вписывается в EHR-FM, здесь

Context Management Specifications (CCOW)

CCOW - это стандарт, позволяющий независимым системам синхронизировать контекст на одной рабочей станции, обеспечивая бесшовный интерфейс для пользователя этой рабочей станции (например обеспечение непротиворечивой аутентификации пользователей, отображение того же самого пациента, отображение того же самого заказа и т. п.). Теоретически, FHIR-ресурсы можно было бы использовать в качестве альтернативной технологии реализации CCOW, однако для этого не очевиден бизнес-кейс. Профили CCOW включают в себя мэппинги на V2. Эти мэппинги могут использоваться для определения эквивалентных элементов данных FHIR при установлении связей CCOW в системах на основе FHIR.

Arden Syntax

Арден синтаксис (Arden Syntax) - это язык для определения правил поддержки принятия решений. Эти правила дают ссылку на элементы данных, которые используются в рамках процесса принятия решений. Однако эта спецификация не задает, как эти элементы данных идентифицируются. Идентификаторы элементов и расширений FHIR обеспечат единый механизм определения релевантных элементов данных.

Virtual Medical Record

Виртуальная медицинская запись - это проект спецификации, разрабатываемый HL7, который также обслуживает область поддержки принятия решений. Он определяет логическую медицинскую запись, относительно которой могут быть построены правила поддержки принятия решений. В настоящее время эта модель является пользовательской моделью, созданной специально для VMR. Тем не менее, рабочая группа поддержки принятия решений оценивает возможность использования FHIR в качестве структуры будущих версий этой спецификации.
