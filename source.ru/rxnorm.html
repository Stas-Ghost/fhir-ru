Сводные данные

SourceRxNorm создан под управлением US National Library of Medicine и доступен по адресу http://www.nlm.nih.gov/research/umls/rxnorm
SystemКодовую систему RxNorm идентифицирует следующий URI: http://www.nlm.nih.gov/research/umls/rxnorm
VersionТам, где используется версия, она должна быть датой релиза, закодированной так же, как и в скачиваемых файлах, например "07092014"
CodeЗначением элемента code для RxNorm-кода будет Concept Identifier (CUI), и только те CUIs, для которых выполняется условие SAB=RXNORM
DisplayКорректным значением элемента display для CUI будет его строковое описание, ассоциируемое с источником RXNORM. Значения элемента Display не чувствительны к регистру

База данных RxNorm MySQL

Скрипты RxNorm могут заполнять базу данных MySQL, которая содержит данные из RxNorm. Здесь приводятся SQL-запросы, описывающие, как правильно реализовать функциональные возможности терминологии RxNorm в этой базе данных. Они созданы для удобства реализаторов и не подразумевают, что в реализациях необходимо использовать какой-то определенный подход.

К примеру, корректным названием для отображения CUI будет 'Select STR from rxnconso where RXCUI = :code and SAB = 'RXNORM' and TTY <> 'SY'.

Вопрос авторских прав и лицензий

Использование кодов и значений для отображения RxNorm в данной спецификации согласуется со статусом HL7 как лицензиата NLM UMLS. Эта лицензия не передает право на использование RxNorm пользователям данной спецификации; реализаторы должны самостоятельно получить лицензию на право использования RxNorm.

Лицензия на RxNorm ставит ограничения на распространение метасправочника UMLS или его подмножеств. HL7 подбирает четкое определение того, что представляет собой "подмножество метасправочника", чтобы дать ясно понять, какой статус имеет набор значений RxNorm.

Свойства фильтра RxNorm

В этом разделе задокументированы свойства фильтров, которые могут использоваться в кодовой системе RxNorm при сравнении наборов значений.

Базовый SQL-запрос для получения списка CUIs, которые соответствуют этим фильтрам: 

  Select RXCUI from rxnconso where SAB = 'RXNORM' and TTY <> 'SY'

Взаимосвязь

ОписаниеПозволяет выбирать концепт на основе его взаимосвязей
Имя свойства[REL]
Допустимые операции= / in
Допустимые значенияCUI:[RXCUI] или AUI:[RXAUI] должен быть допустимым CUI или AUI. Обратите внимание, что CUI не требуется иметь запись SAB=RXNORM, чтобы использоваться здесь
Комментарии[REL] (:rel) - это один из SY, SIB, RN, PAR, CHD, RB или RO
SQL для CUI: 
and (RXCUI in (select RXCUI from rxnconso where RXCUI in (select RXCUI1 from rxnrel where REL = :rel and RXCUI2 = :value))
для AUI: 
and (RXCUI in (select RXCUI from rxnconso where RXAUI in (select RXAUI1 from rxnrel where REL = :rel and RXAUI2 = :value))

Тип взаимосвязи

ОписаниеПозволяет выбирать концепт на основе типа его взаимосвязи
Имя свойства[RELA]
Допустимые операции= / in
Допустимые значенияCUI:[RXCUI] или AUI:[RXAUI] должен быть допустимым CUI или AUI. Обратите внимание, что CUI не требуется иметь запись SAB=RXNORM, чтобы использоваться здесь
Комментарии[RELA] (:rela) - это один из типов взаимосвязи, приведенных ниже
SQL для CUI:

для AUI: 
and (RXCUI in (select RXCUI from rxnconso where RXAUI in (select RXAUI1 from rxnrel where RELA = :rel and RXAUI2 = :value))

Имплицитные наборы значений

Имплицитные наборы значений - те, спецификация которых может быть предсказана на основе грамматики лежащей в основе кодовой системы и известной структуры URL, который ссылается на них. На момент данной публикации RxNorm ещё не определил имплицитные наборы значений.

Идентификатор http://www.nlm.nih.gov/research/umls/rxnorm/vs указывает на набор значений, который содержит все CUIs RxNorm.
