Эталонные реализации
В помощь разработчикам были созданы эталонные реализации стандарта. "Эталонные" FHIR-серверы могут использоваться на практике в клинических целях, однако организация HL7 и разработчики реализаций снимают с себя всю ответственность за возможные последствия. Эти реализации предоставляются по стандартной, одобренной OSI лицензии (пункт 3 BSD).

Обратите внимание, что эталонные реализации представлены для помощи в реализации спецификации и поддерживаются командой проекта FHIR, но не являются частью спецификации, реализациям не требуется соответствовать им, и они не подлежат официальному продвижению стандартов.

Другие эталонные реализации, не распространяемые вместе со спецификацией:

  • Swift-FHIR - эталонная реализация для Swift
  • HAPI-FHIR - эталонная реализация для Java

Следует отметить, что эталонные реализации обычно ограничиваются визуальным представлением содержимого ресурсов в нативной форме, и их парсингом и сериализацией в XML и JSON. Некоторые из этих реализаций поддерживают построение с помощью определений ресурсов. В некоторые реализации входит клиентское приложение, которое поддерживает RESTful API.

Полноценные открытые реализации FHIR, некоторые из которых используют эти базовые реализации, перечислены в wiki-разделе сайта HL7.

Для обеспечения соответствия не обязательно пользоваться этими конкретными реализациями. Разрешается применять любые подходы, включая генерацию кода из схем.
