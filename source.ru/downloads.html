Формальные определения
Schema XML-схемы для FHIR предлагаются в двух видах:
  • Схемы валидации (включают схемы поддержки, схемы ресурсов, агрегатные и комбинированные схемы, а также схематрон-схемы
  • Схемы кодогенерации (см. примечания к схемам кодогенерации)
Примечание: схемы и схематрон-схемы не содержат всех правил валидности ресурсов. Для создания соответствующих спецификации реализаций всё равно необходимо знакомство с текстом спецификации и с профилями, которые применяются к ресурсам.
Определения FHIR Сюда входят все наборы значений, профили и т. п., определенные в рамках спецификации FHIR, и включённые в спецификацию руководства по реализации. Эти определения представлены в 4 вариантах - XML или JSON, с или без текстовых определений:
  • XML: с текстом и без текста
  • JSON: с текстом и без текста
Обратите внимание, что варианты без определений включены для применения с инструментами, которые не используют текстовые определения (например инструменты валидации) - они меньше и грузятся быстрее.
Примеры Все примеры ресурсов представлены в виде zip-файлов. Их можно использовать для инициализации тестовых репозиториев и т. п.

  • JSON-формат

Эталонные реализации
Эталонные реализации предоставлены в помощь разработчикам. Их можно использовать в реально действующих экземплярах, однако организация HL7 и её сотрудники не несут ответственность за их применение. Эти реализации предоставляются по стандартной, одобренной OSI лицензии (по большей части пункт 3 BSD).

Обратите внимание, что эталонные реализации предоставлены для помощи реализаторам в адаптации спецификации и поддерживаются командой проекта FHIR, но не являются частью спецификации, и от реализаций не требуется соответствовать им, также они не подлежат формальному развитию стандарта.

Другие эталонные реализации, не распространяемые вместе со спецификацией:

  • Swift-FHIR - эталонная реализация для Swift
  • HAPI-FHIR - эталонная реализация для Java

Следует отметить, что эталонные реализации обычно ограничиваются визуальным представлением содержимого ресурсов в нативной форме, их парсингом и сериализацией в XML и JSON. Часть реализаций поддерживает построение с помощью определений ресурсов. В некоторые реализации входит клиентское приложение, которое поддерживает RESTful API.

Полноценные открытые реализации FHIR, часть которых использует эталонные реализации, перечислены в wiki-разделе сайта HL7.

Для обеспечения соответствия не обязательно пользоваться этими конкретными реализациями. Разрешается применять любые подходы, включая генерацию кода из схем.
