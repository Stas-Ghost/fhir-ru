Схемы (Schema) Все схемы, собранные в zip-архиве (дополнительные схемы (support schemas), схемы ресурсов, схемы модулей, комбинированные схемы и схематроны (schematron))
Набор для валидации (Validation Pack) Валидатор FHIR (java jar) и все необходимое для его работы (включая нужную версию Saxon)
Файл перевода (Translation File) Перевод наиболее часто используемых имен и сообщений FHIR на различные языки (В wiki-разделе Вы можете найти информацию, как можно сделать личный вклад в перевод имен и сообщений FHIR
Спецификация FHIR в книжном формате (FHIR Book) Спецификация FHIR в формате электронной книги (epub). Программы для чтения epub: PC, OSX, iOS, Android, Linux
Спецификация FHIR (FHIR Specification) Полная спецификация FHIR для размещения на вашем локальном сервере (исключен раздел "Загрузки")
Профили комбинированных ресурсов (Combined Resource Profiles) Единый агрегатор ресурсов (feed), содержащий описания всех ресурсов . Данные профили могут быть полезны, если Вы только начинаете создавать собственные профили ресурсов или при составлении Заявления о соответствии (Conformance Statement)
Все подмножетсва значений (All valuesets) Все подмножества значений, определенные как часть стандарта FHIR
Примеры ресурсов в формате XML (XML Examples) Примеры всех ресурсов в XML-представлении (zip-архив)
Примеры ресурсов в формате JSON (JSON Examples) Примеры всех ресурсов в JSON-представлении (zip-архив)
EMF Core An EMF Core model of the resources (using EMFText)
Пиктограммы FHIR (Icon Pack) Пиктограмма FHIR в различных разрешениях. В любых программных реализациях стандарта FHIR от организаций-участниц конференций по интеграции (Connectahon) разрешается использовать пиктограммы FHIR. Однако, данная политика будет пересмотрена в будущем.
Эталонные реализации (Reference Implementations)
В помощь разработчикам были созданы эталонные реализации стандарта. "Эталонные" FHIR-серверы могут использоваться в практических клиничсеких целях, однако, организация HL7 и разработчики реализаций снимают с себя всю ответственность за возможные последствия. Эти реализации предоставляются по стандартной, одобренной OSI лицензии (пункт 3 BSD).

Note that these reference implementations are provided to assist to implementers to adopt the specification, and are maintained by the FHIR project team, but are not part of the specification, and implementations are not required to conform to these, nor are they subject to the formal standards process.

Заметьте: эталонные реализации обычно имеют ограниченные возможности использования кодирования для представления содержимого ресурсов в нативной форме, а также ограниченные возможности парсинга и сериализации XML / JSON. Некоторые реализации поддерживают построение и использование ресурсов с помощью их определений (resource definition), а также имеют встроенного клиента, который соответствует принципам REST (conforms to the RESTful API).

Полноценные открытые реализации FHIR, некоторые из которых основаны на этелонной, перечислены в wiki-разделе сайта HL7.

Для обеспечения соответствия не обязательно пользоваться данными конкретными реализациями. Разрешается применять любой подход, включая генерацию кода из схем.
