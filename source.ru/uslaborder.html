Реализация US Laboratory Order

Реализация US Laboratory Order (USLabOrder) состоит из документации по руководству, профилей, операций, наборов значений и заявлений о соответствии, которые она определяет. Данная реализация была подготовлена как U.S. Realm спецификация от имени рабочей группы HL7 Orders and Observation.

Заказ лабораторных исследований в США для случая амбулаторной помощи (USLabOrder) - Этот сценарий использования основан на существующих законных требованиях к Лабораториям и Системам электронных медицинских записей (EHR-S) для заказа клинических лабораторных исследований. Содержимое было смоделировано после объединения с HL7, The Standards and Interoperability (S&I) Framework Laboratory Orders Interface (LOI) Initiative и HL7 Lab Order Conceptual Specification. Однако большинство материала, вероятно, будет полезно и помимо амбулаторных случаев, и в других странах.

Спецификация

USLabOrder построен поверх стандарта HL7 FHIR. В нем применяются основные аспекты протокола FHIR, включая RESTful-операции, типы данных, поиск и др.

Profiles - на данный момент реализация USLabOrder сосредоточена на использовании FHIR-ресурса Bundle для обмена DiagnosticOrder Profile и профилей ресурсов, на которые он ссылается. Этот FHIR bundle полностью определяет лабораторный заказ. Профили ресурсов, которые могут использоваться в комплекте, описываются ниже: (Этот список не исчерпывающий, так как дополнительные ресурсы также могут использоваться)

  • US Lab DiagnosticOrder ограничивает и расширяет DiagnosticOrder, который используется для определения лабораторного заказа. Этот ресурс ссылается на следующие профили:
    • USlabPatient ограничивает и расширяет Patient, который используется для задания субъекта заказа.
    • USlabPHPatient ограничивает и расширяет Patient, который используется для задания субъекта заказа в тех сценариях использования, где необходима дополнительная информация о пациенте, например для исследований, которые могут заслуживать публикации для общественного здравоохранения, и когда заказчик поручает лаборатории отправить копию результатов пациенту.
    • USlabPractitioner ограничивает и расширяет ресурс Practitioner, который используется для задания провайдера, который заказал лабораторное исследование(-ия), или не оформлявший заказ провайдер, которому заказчик поручил лаборатории выслать результаты.
    • USlabPHPractitioner ограничивает и расширяет ресурс Practitioner, который определен для случаев использования, где требуется дополнительная информация о провайдере, например для исследований, которые могут заслуживать публикации для общественного здравоохранения. Этот ресурс ссылается на следующие профили:
      • USLabOrderFacility ограничивает и расширяет ресурс Organization, который используется для задания организации заказывающего.
    • USLabObsCode | USLabObsQuantity | USLabObsOther | USLabObsRatio ограничивают и расширяют ресурс Observation, который используется для поддержки различных типов результатов для записи вспомогательной информации или других клинически релевантных наблюдений, например предыдущих лабораторных результатов и вопросов, задаваемых при заказе (ask at order questions (AOE)). Эти ресурсы ссылаются на следующий профиль:
      • USLabPerformingLab ограничивает и расширяет ресурс Organization, который используется для задания организации, которая несет ответственность за это наблюдение.
    • USlabReasonForStudy ограничивает и расширяет ресурс Condition, который используется для записи причин выполнения заказанного исследования.
    • USlabCond ограничивает и расширяет ресурс Condition, который используется для предоставления другой клинически релевантной информации лаборанту или патологоанатому.
    • USlabSpec ограничивает и расширяет ресурс Specimen, который используется для определения того, что, когда и как было взято в качестве образца.
    • USLabCCTarget ограничивает и расширяет ресурс Organization, который используется для определения организации третьей стороны, которой заказчик поручает лабораторию выслать результаты.
