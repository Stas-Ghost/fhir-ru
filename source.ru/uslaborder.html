USLabOrder Implementation Guide

The USLabOrder Implementation Guide consists of the IG, the profiles, operations, value sets and conformance statements it defines. This implementation guide is prepared as a U.S. Realm Specification on behalf of the HL7 Orders and Observation Workgroup. Its scope is US Realm ambulatory care and is based upon upon existing regulatory requirements for Laboratories and Electronic Health Record Systems (EHR-S) for ordering clinical laboratory results. The content has been modeled after the joint HL7 and The Standards and Interoperability (S and I) Framework Laboratory Orders Interface Initiative and the HL7 Lab Order Conceptual Specification . However, much of the content is likely to be useable outside the ambulatory space and in other jurisdictions.

Draft for Comment Ballot

This guide will be balloted together with the FHIR specification itself as part of the second FHIR DSTU as part of the December-January ballot cycle. This will provide an opportunity to provide feedback on all aspects of the FHIR specification.

Specification

The USLabOrder Implementation Guide is built on top of the HL7 FHIR standard. Basic aspects of the FHIR protocol, including RESTful operations, data types, search, etc. apply.

It makes use of the following resources:

  • Attachment is used to provide PDF and CDA attachments for supplying previous results or other clinically relevant information to the laboratorian or pathologist.
  • Condition is used to record reasons for performing the requested test and possibly other clinically relevant information to the laboratorian or pathologist.
  • Conformance is used to define additional constraints in the lab order
  • DiagnosticOrder is used to define the laboratory order
  • DiagnosticReport is used to define the laboratory results report in the case of prior results
  • Observation is used to record individual test measurements for ask at order questions (AOE) and in the case of prior results
  • Organization is used to define the provider's office, clnic, lab or hospital.
  • Media is used to provide images for supplying previous results
  • Patient is used to define the patient
  • Practitioner is used to define the provider who ordered the lab test(s)
  • Specimen is used to define the what when and how the specimen was collected
  • Value Set is used to define allowed values for data elements in the lab order

This list is not meant to be exhaustive as additional resources may also to be used in USLabOrder solutions.

The USLabOrder specification consists of the following components:

  • USLabOrder profiles - Profiles on the eight FHIR resources used to support the IG requirements:
    • ConditionConstraints and extensions for providing condition information for USRealm laboratory ordering
    • DiagnosticOrderConstraints and extensions for the USRealm laboratory ordering
    • DiagnosticReport Constraints and extensions for USRealm laboratory results reporting
    • Observation Constraints and extensions for ask at order questions (AOE) and US Realm laboratory test observations
    • PatientConstraints and extensions to the patient resource for USRealm laboratory ordering and resulting
    • PractitionerConstraints and extensions to the practitioner resource for USRealm laboratory ordering and resulting
    • OrganizationConstraints and extensions to define the provider's office, clnic, lab or hospital for USRealm laboratory ordering and resulting.
    • Specimen Constraints and extensions to define the specimen attributes for for USRealm laboratory ordering
