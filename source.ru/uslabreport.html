Реализация лабораторного отчета в США

Реализация US Laboratory Report Implementation (USLabReport) состоит из документации по руководству, профилей, операций, наборов значений и заявлений о соответствии, которые она определяет. Данная реализация была подготовлена в виде спецификации для США от имени рабочей группы HL7 Orders and Observation.

Составление отчетов по лабораторным исследованиям в США для случая амбулаторной помощи (USLab Report) - Этот сценарий использования основан на существующих нормативных требованиях к Лабораториям и Системам электронных медицинских записей (EHR-S) для составления отчетов клинико-лабораторных исследований. Содержимое было смоделировано после объединения с HL7, The Standards and Interoperability (S&I) Framework Laboratory Results Interface (LRI) Initiative and the HL7 V3 Lab Normative Standard . Однако большинство материала будет, вероятно, полезно и помимо амбулаторных случаев, а также для других стран.

Спецификация

USLabReport построен поверх стандарта HL7 FHIR. В нем применяются основные аспекты протокола FHIR, включая RESTful-операции, типы данных, поиск и др.

Profiles - на данный момент реализация USLabReport сосредоточена на использовании FHIR-ресурса Bundle для обмена DiagnosticReport Profile и профилей ресурсов, на которые он ссылается. Этот FHIR bundle полностью определяет лабораторный заказ. Профили ресурсов, которые могут использоваться в комплекте, описываются ниже: (этот список не исчерпывающий, так как дополнительные ресурсы также могут использоваться)

  • US Lab DiagnosticReport ограничивает и расширяет ресурс DiagnosticReport, который используется для задания отчета с лабораторными результатами. Этот ресурс ссылается на следующие профили:
    • USlabPatient ограничивает и расширяет ресурс Patient, который используется для задания субъекта заказа.
    • US Lab DiagnosticOrder ограничивает и расширяет ресурс DiagnosticOrder, который используется для определения лабораторного заказа. Этот ресурс ссылается на эти USLabOrder профили:
    • USlabSpec ограничивает и расширяет ресурс Specimen, который используется для определения того, что, когда и как было взято в качестве образца.
    • USLabObsCode | USLabObsQuantity | USLabObsOther | USLabObsRatio ограничивают и расширяют ресурсы Observation, которые используются для поддержки различных типов результатов для записи лабораторных наблюдений.
      • USLabPerformingLab ограничивает и расширяет ресурс Organization, который используется для задания организации, которая несет ответственность за это наблюдение.

Помимо задания комплекта (bundle) USLabReport, данная реализация состоит из следующих компонентов:

  • Operations - некоторая USLabReport операция - Вопрос для голосования: есть ли какие-то операции, которые должен рассмотреть USLabReport?

    • USLabReport заявление о соответствии получателя
  • Value Sets - USLab Value Sets перечисляет дополнительные привязки Value Set, созданные для USLabOrder и USLabReport реализаций.
