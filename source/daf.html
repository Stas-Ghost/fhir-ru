Data Access Framework (DAF) Implementation Guide

This implementation guide is prepared as a U.S. Realm Specification on behalf of the U.S. Office of the National Coordinator (ONC). However, much of the content is likely to be usable in other jurisdictions.

Draft for Comment Ballot

This FHIR Implementation Guide is undergoing a Draft for Comment ballot. The focus of the ballot is limited in scope to the IG, the profiles, operations, value sets and conformance statements it defines, though comments on the underlying resources will also be accepted. This guide will be balloted seperately from the FHIR specification itself.

Specification

The DAF FHIR Implementation Guide is built on top of the HL7 FHIR standard. It makes use of the following profiles and resources:

  • DAF Allergy Intolerance profile using the AllergyIntolerance resource.
  • DAF Cognitive Status profile using the Observation resource.
  • DAF Encounter profile using the Encounter resource.
  • DAF Family History profile using the FamilyHistory resource.
  • DAF Immunization profile using the Immunization resource.
  • DAF Lab Results profile using the Observation resource.
  • DAF Medication Administration profile using the MedicationAdministration resource.
  • DAF Medication Dispense profile using the MedicationDispense resource.
  • DAF Medication Prescription profile using the MedicationPrescription resource.
  • DAF Patient profile using the Patient resource.
  • DAF Problems (Condition) profile using the Condition resource.
  • DAF Procedure profile using the Procedure resource.
  • DAF Smoking Status profile using the Observation resource.
  • DAF Vital Signs profile using the Observation resource.
  • Medication
  • DiagnosticOrder
  • DiagnosticReport
  • Practitioner
  • List

Additional resources such as Provenance, SecurityEvent and others are likely to be used in DAF solutions. As well, basic aspects of the FHIR protocol, including RESTful operations, data types, search, etc. also apply.

The DAF Conformance statements which define the expected capabilities of each of the DAF actors are listed below:

  • DAF Requestor - DAF Requestors are responsible for creating and initiating the queries for patients or populations.
  • DAF Responder- DAF Responders are responsible to provide responses to the queries submitted by the requestors.
