Таблица статус-кодов FHIR

This is an internal QA page. The intent of this table is to help reviewers compare the mappings between the general resource status codes and the status codes on particular resources.

Примечание: данная страница не является запретительной; она описательная и сравнительная для помощи в рецензировании. Методология FHIR говорит, что если два домена используют разные термины для одного и того же концепта (например 'withdrawn' and 'cancelled'), тогда следует использовать доменное имя концепта. Наоборот, если явная причина различия кодов на основе доменного использования отсутствует, они не должны различаться.

All status codes must be mapped to one of the general status codes, which are found in the Resource Status code system.

Примечание: Bold показывает, что элемент является модификатором.
